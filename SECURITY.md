# Security Policy

If you discover a security issue in Blueprint, report it privately.

Include:

- a short summary
- affected route, module, or Discord workflow
- reproduction steps
- expected impact
- any logs or screenshots that help validate the issue

Do not disclose exploitable issues publicly before the maintainers have had a reasonable chance to investigate and fix them.

For live deployments, the canonical machine-readable reporting endpoint is:

- `/security.txt`
- `/.well-known/security.txt`

Because this project may be deployed in different environments, the exact contact path is published by the running app rather than hardcoded only in the repository.
